# SEU-Bus-time
这是东南大学的校车时刻表，可以查看不同线路校车的发车时间。

# 校车时间查询系统

一个纯前端、开箱即用的校车实时查询工具，支持工作日/节假日自动切换、实时倒计时、完整时刻表浏览，无需后端服务器，单 HTML 文件即可运行。

## 功能特性

- ✅ 智能识别工作日/节假日，自动加载对应时刻表
- ✅ 手动强制切换时刻表模式
- ✅ 实时倒计时（精确到秒），显示下一班及再下一班发车时间
- ✅ 发车状态智能判断（即将发车/准备发车/等待发车/已发车/服务结束）
- ✅ 校内接驳车高峰时段（限行时间）橙色高亮提示
- ✅ 兰台2号线早高峰循环发车动态提示
- ✅ 全部5条线路实时查询与线路筛选
- ✅ 完整时刻表浏览页，支持线路筛选
- ✅ 响应式设计，适配手机和电脑
- ✅ 每秒自动刷新，手动刷新按钮
- ✅ 无任何外部依赖，离线可用
- ✅ 无后门、无数据收集、完全透明安全

## 使用方法

### 1. 模式选择

页面顶部提供三个按钮：
- **自动**：根据当前系统日期自动判断（周一至周五为工作日，周六周日为节假日）
- **工作日**：强制使用工作日时刻表
- **节假日**：强制使用节假日时刻表

选择后，所有班车信息和倒计时将按所选模式更新，提示条背景色会相应变化（浅绿色=工作日，浅红色=节假日）。

### 2. 首页实时查询

点击底部标签栏的 **首页** 进入实时查询页面。

- **线路筛选**：点击“全部校车”或具体线路按钮（校内接驳车、无线谷1号线、无线谷2号线、兰台1号线、兰台2号线），页面仅显示所选线路
- **查看班车信息**：每条线路的两个行驶方向分别展示：
  - 发车倒计时（精确到秒，格式如 `00:02:30`）
  - 下一班发车时间点
  - 再下一班发车时间点（如有）
  - 当前状态标签（即将发车/准备发车/等待发车/已发车/今日服务结束）
- **特殊提示**：
  - 校内接驳车高峰时段发车时间为橙色，并显示“橙色为限行时间”
  - 兰台2号线早高峰时段自动显示“循环发车”提示
- **刷新**：倒计时每秒自动更新，也可点击“刷新”按钮立即强制刷新

### 3. 时刻表浏览

点击底部标签栏的 **时刻表** 进入完整时刻表页面。

- **线路筛选**：点击“全部线路”或具体线路按钮，仅显示所选线路的时刻表
- **时刻表展示**：每条线路按方向分组，所有发车时间以“时间气泡”形式排列
- **响应式布局**：手机端每行显示3个时间气泡，电脑端自适应显示更多
- 同样保留高峰期橙色高亮和循环发车提示

### 4. 退出程序

直接关闭浏览器标签页或应用窗口即可。所有手动设置（如强制切换的时刻表模式、线路筛选）不会自动保存，下次打开将恢复为默认的自动模式。

## 数据维护

如需修改校车时刻表，请直接编辑代码中的以下两个数据对象：

- `workdayBusData`：工作日班车数据
- `holidayBusData`：节假日班车数据

每个对象的结构为：线路ID → 线路名称 → 方向名称 → 停靠站点 → 发车时间数组。修改后保存文件即可生效。

## 常见问题

**问：为什么倒计时显示“已发车”但实际校车还没到？**  
答：倒计时严格按时刻表中的发车时间计算，实际校车到站时间可能因路况等因素略有延迟，建议提前到达站点等候。

**问：如何修改高峰时段？**  
答：编辑代码中的 `peakHours` 和 `peakHours2` 数组，按 `"HH:MM"` 格式添加或删除时间点。

**问：能否添加新的线路？**  
答：可以。在 `workdayBusData` 和 `holidayBusData` 中按现有结构添加新的线路ID及对应时刻表数据即可。

**问：手动选择的时刻表模式会保存吗？**  
答：不会。每次刷新页面或重新打开软件，都会恢复为“自动”模式。如需持久化保存，可修改代码增加 localStorage 存储功能。
